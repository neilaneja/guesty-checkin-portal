Guesty Check-In Portal — Setup Guide

This guide walks you through deploying your self-serve check-in portal step by step. No coding required — just follow the instructions and fill in your info. Total time: about 30 minutes.

1
5 minutes

Create a GitHub Account

GitHub is where your code will live. Railway (the hosting service) pulls from it automatically.

  1. Go to github.com and click "Sign up"
  2. Use your email, create a password, and pick a username
  3. Verify your email when they send you a confirmation
If you already have a GitHub account, skip to Step 2.
2
5 minutes

Upload the Code to GitHub

  1. Log into GitHub
  2. Click the + icon (top right) → "New repository"
  3. Name it guesty-checkin-portal
  4. Keep it Private
  5. Click "Create repository"
  6. On the next page, click "uploading an existing file" (it's a link in the instructions)
  7. Drag and drop all the files from the guesty-checkin-portal folder into the upload area
  8. Click "Commit changes"
Important: Make sure you upload the folder contents, not the folder itself. You should see files like server.js, package.json, and folders like api/, public/, config/ at the top level.
Don't upload the .env.example file (it's just a template). Your actual secrets will go into Railway in Step 4.
3
5 minutes

Deploy on Railway

Railway hosts your app and makes it available on the internet.

  1. Go to railway.app
  2. Click "Login" → choose "Login with GitHub"
  3. Authorize Railway to access your GitHub
  4. Click "New Project"
  5. Select "Deploy from GitHub Repo"
  6. Find and select guesty-checkin-portal
  7. Railway will automatically detect it's a Node.js app and start deploying
Railway gives you a free tier with $5/month of usage — more than enough for a check-in portal that only gets traffic when guests arrive.
4
10 minutes

Get Your Guesty API Credentials & Add Them to Railway

Part A — Get your Guesty API keys:

  1. Log into your Guesty Dashboard
  2. Go to Marketplace (left sidebar) → Open API
  3. Create a new API integration (or use existing)
  4. Copy your Client ID and Client Secret

Part B — Get your Listing IDs:

  1. In Guesty, go to Listings
  2. Click on a listing
  3. Look at the URL in your browser — the ID is the long string after /listings/
  4. Copy this ID for each property

Example URL:

https://app.guesty.com/listings/64a1b2c3d4e5f6a7b8c9d0e1/details

The bold part is your Listing ID.

Part C — Add secrets to Railway:

  1. In Railway, click on your deployed service
  2. Go to the "Variables" tab
  3. Click "New Variable" and add these two:
GUESTY_CLIENT_ID = your_client_id_here GUESTY_CLIENT_SECRET = your_client_secret_here
  1. Railway will automatically redeploy with the new variables
5
5 minutes

Configure Your Properties

Before deploying, you need to edit config/properties.json in your GitHub repo with your actual property details.

  1. Go to your GitHub repository
  2. Click on config → properties.json
  3. Click the pencil icon (edit) in the top right
  4. Replace the example data with your real property info:
// For each property, fill in: { "slug": "west-end-flats", // URL-safe name (no spaces) "name": "The West End Flats", // Display name guests will see "welcomeMessage": "Welcome!", // Greeting on the check-in page "logoUrl": "/logos/my-logo.png", // or leave blank "" "brandColor": "#2C3E50", // Header color (hex) "accentColor": "#E67E22", // Button color (hex) "fallbackPhone": "(555) 123-4567", // Emergency contact "guestyListingIds": ["your_listing_id"] // From Step 4B }
  1. Click "Commit changes" — Railway will auto-redeploy
Pick colors: Use google.com/search?q=color+picker to find hex color codes you like. The brand color is used for headings, the accent color is used for the search button.
6
5 minutes

Connect Your Custom Domain

Railway gives you a URL like guesty-checkin-portal.up.railway.app, but you want guests to see your own domain.

  1. In Railway, click your service → "Settings" tab
  2. Scroll to "Custom Domain"
  3. Type your domain (e.g., thewestendflats.com) and click Add
  4. Railway shows you a CNAME record to add
  5. Go to wherever you bought your domain (GoDaddy, Namecheap, Cloudflare, etc.)
  6. Find DNS Settings
  7. Add a new CNAME record:
Type: CNAME Name: @ (or leave blank — this means "the root domain") Value: [the value Railway gave you]
  1. Wait 5–30 minutes for DNS to propagate
  2. Railway handles SSL (the lock icon) automatically
If your domain provider doesn't support CNAME on the root domain (some don't), use a subdomain like checkin.thewestendflats.com instead. Or switch your DNS to Cloudflare (free) which does support it.
7
5 minutes

Verify the Guesty Check-In Form URL

This is an important step. The app redirects guests to their Guesty check-in form, but the exact URL pattern can vary by account.

  1. In Guesty, go to Operations → Guest App
  2. Find a test reservation and look at its check-in form link
  3. The URL will look something like:
    https://app.guestybooking.com/guest-app/RESERVATION_ID
  4. If your URL looks different, you'll need to update one line in server.js
If you're not sure, send me (Neil) a screenshot of what the check-in form URL looks like in your Guesty dashboard, and I can tell you if the code needs to change.
8
2 minutes

Test It

  1. Open your domain in a browser: yourdomain.com/checkin/your-property-slug
  2. You should see the branded check-in page with the last name field
  3. Enter a last name that matches a current reservation in Guesty
  4. You should be redirected to the Guesty check-in form
If "Reservation not found": Make sure the listing IDs in properties.json are correct, and that the test reservation's check-in date is within 48 hours of right now.
9
5 minutes

Print Your QR Code Signs

  1. Go to: yourdomain.com/signage/your-property-slug
  2. The form auto-fills with your property info
  3. Click "Generate Sign" to preview
  4. Click "Print"
  5. Print on cardstock or regular paper, then laminate
  6. Place at each property entrance near the door
Pro tip: Print a few extras. Weather and wear will degrade outdoor signs. Consider placing one at the front door and one inside the entry area as a backup.

Final Checklist

  • GitHub account created
  • Code uploaded to GitHub repository
  • Railway account created and linked to GitHub
  • Guesty API credentials added to Railway variables
  • properties.json updated with real property info and listing IDs
  • Custom domain connected and SSL working
  • Guesty check-in form URL verified
  • Test check-in completed successfully
  • QR code signs printed and placed at properties
